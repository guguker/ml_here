# GeoPredict: юридический чеклист перед публикацией

Это технический чеклист, а не юридическое заключение.

## Реализовано

- отдельное пользовательское соглашение;
- отдельная политика обработки персональных данных;
- отдельное согласие на обработку персональных данных;
- две независимые галочки при регистрации;
- backend отклоняет регистрацию без всех согласий;
- PostgreSQL хранит пользователя, версии документов, дату принятия, IP и User-Agent;
- создание аккаунта и записи согласия выполняется одной транзакцией;
- в документах описаны email, сессии, технические журналы, OSM и сроки хранения;
- указан порядок отзыва согласия и запроса удаления данных.

## Обязательно заполнить до публичного запуска

1. Точный адрес регистрации ИП из ЕГРИП. Сейчас улица Кирова указана только как
   явно помеченный шаблон без выдуманного номера дома.
2. ИНН и ОГРНИП Зотова Вадима Кирилловича.
3. Фактические сведения о российском хостинге PostgreSQL и первичной записи
   персональных данных.
4. Перечень инфраструктурных подрядчиков и условия передачи им данных.
5. Реальные сроки хранения серверных журналов.
6. Порядок ответа на запросы субъектов персональных данных.

## Организационные действия

- до начала обработки проверить обязанность подать уведомление оператором
  персональных данных в Роскомнадзор;
- назначить ответственного за обработку персональных данных;
- утвердить внутреннюю политику доступа, удаления, резервного копирования и
  реагирования на инциденты;
- включить HTTPS и безопасные cookie в production;
- хранить секреты JWT и базы данных только в переменных окружения;
- заключить необходимые договоры с хостингом и иными обработчиками;
- провести финальную проверку документов профильным юристом.

## Админ-панель

Наличие визуальной админ-панели законом не требуется. Для первой версии
достаточно защищённых административных операций: найти пользователя,
заблокировать или удалить аккаунт, отозвать сессии и проверить журнал согласий.
Полный интерфейс следует добавлять после реализации ролей, аудита действий,
многофакторной аутентификации и серверной авторизации каждого действия.
